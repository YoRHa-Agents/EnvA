EnvA

「 System Capabilities 」

「 Features 」

◈

AES-256-GCM Vault

Secrets are encrypted at rest with AES-256-GCM. Key derivation uses Argon2id with tunable parameters. HMAC-SHA256 verifies vault integrity on every load.

◈

App-Aware Injection

Assign secrets to specific applications. Each app receives only its designated environment variables at runtime via process-level injection.

◈

Built-in Web UI

Browse, edit, rename, assign, import, and export secrets through a clean embedded web interface. No external dependencies required.

◈

SSH Remote Sync

Read SSH config hosts, preview remote vault contents, and run selective deploy or sync operations without leaving the web UI.

◈

Self-Update

Built-in updater fetches the latest release from GitHub, verifies SHA256 digest, and atomically replaces the running binary in place.

◈

Cross-Platform

Pre-built binaries for Linux x86_64, Linux aarch64, and macOS Apple Silicon. Single static binary with zero runtime dependencies.

「 Deployment Protocol 」

「 Installation 」

curl -fsSL https://raw.githubusercontent.com/YoRHa-Agents/EnvA/main/scripts/install.sh | bash
git clone https://github.com/YoRHa-Agents/EnvA.git && cd EnvA
cargo build --release
sudo cp target/release/enva /usr/local/bin/
enva vault self-test
enva update --help
「 Operational Guide 」

「 Quick Start 」

1

Initialize a Vault

Create an encrypted vault file to store your secrets.

enva vault init --vault ./my.vault.json
2

Store a Secret

Add a secret with an alias, key name, and value.

enva vault set db-url -k DATABASE_URL -V "postgres://user:pass@host/db"
3

Assign to an App

Bind the secret to a specific application context.

enva vault assign db-url --app backend
4

Inject and Run

Launch your application with secrets injected as environment variables.

enva backend -- printenv DATABASE_URL
5

Web UI

Open the built-in web interface for visual management.

enva serve --port 3000
